ABSTRACT
 The Project “Johi Apparels” is a full-stack consultancy application developed to enhance and streamline the operations of apparel businesses. The system integrates three core modules—admin panel, backend, and frontend—to provide a unified platform for managing business processes efficiently. The admin module enables centralized control over users, system settings, and key business metrics, while the backend handles secure data processing, database management, and service integrations such as email notifications and PDF generation. The frontend offers a responsive and user-friendly interface designed to improve customer engagement through interactive components and seamless navigation.
The application leverages modern technologies including React, Node.js, Express.js, and MongoDB to ensure scalability, performance, and maintainability. Additionally, it incorporates analytics features that track user interactions and generate insights to support data-driven decision-making. By automating routine tasks and providing real-time information, Johi Apparels reduces operational complexity and enhances overall productivity.
This project demonstrates the practical implementation of a scalable web-based solution tailored for the apparel industry, focusing on improving efficiency, user experience, and business intelligence.

SYSTEM DESIGN
3.1 SYSTEM ARCHITECTURE
The system is designed using a modular architecture that divides the application into three main components: the admin module, the backend module, and the frontend module. This structure ensures maintainability, scalability, and efficient communication between different parts of the system.
The admin module is responsible for managing the overall system. It allows administrators to monitor key metrics, configure settings, and manage users. This module serves as the control center, providing tools and insights for effective system management.
The backend module handles all server-side operations, including data processing, database management, and integration with external services. It provides secure API endpoints for communication with the frontend and admin modules. Services such as email notifications, PDF generation, and analytics are managed through this module to support automation and intelligent functionality.
The frontend module provides the user interface and interacts with the backend through APIs. It offers a responsive and interactive environment for users to browse products, view information, and engage with the system. User actions on the frontend are tracked and sent to the backend for processing and analytics.
Overall, this three-tier architecture ensures a clear separation of concerns, making the system modular, scalable, and easier to maintain. Each module communicates through defined interfaces, enabling efficient data flow, real-time updates, and a seamless user experience across all platforms.

Fig 3.1	- System Architecture Diagram

3.2 DATABASE DESIGN
The system uses a structured database to store and manage all the information required for its operations. The database is designed using a document-oriented approach with collections to organize data efficiently. Key collections include users, products, orders, quotes, and analytics data. The users collection stores information about administrators and regular users, including credentials and access levels. The products collection contains details about each product, such as name, description, price, quantity, and category. The orders and quotes collections manage transactional data, including order status, quotation details, and customer information. The analytics collection captures data related to user interactions, product views, and other metrics used for generating insights and reports.
Relationships between collections are carefully structured to ensure data integrity and reduce redundancy. For example, orders and quotes reference users and products to maintain consistent linkage between customers and their transactions. Indexing is applied to frequently queried fields to improve performance and allow faster retrieval of information. The database is also designed to scale as the volume of data increases, making it suitable for growing business operations. By using a well-structured database, the system ensures reliable data storage, efficient query execution, and smooth integration with the backend and frontend modules.

Fig 3.2	- Database Entity Relationship Diagram
 
3.3 WORKFLOW DESIGN
The system is designed with a clear workflow to ensure smooth and efficient operation across all modules. The process begins with the user or customer interacting with the frontend module, where they can browse products, view details, request quotes, or place orders. User actions are sent to the backend through API calls, where the data is processed and stored in the database. The backend also handles additional services such as generating PDFs for quotes or orders, sending email notifications, and updating analytics for reporting purposes.
Simultaneously, the admin module allows administrators to monitor system activities, manage users, configure settings, and review business metrics. Information from the frontend and backend is consolidated in the admin module to provide a centralized view of operations. Analytics data is continuously collected and processed to generate insights on user behavior, product performance, and overall system efficiency.
The workflow is designed to ensure that each module communicates seamlessly with others, enabling real-time updates and accurate tracking of business operations. By following this workflow, the system maintains data consistency, reduces errors, automates routine tasks, and provides a smooth experience for both administrators and users. This structured workflow also allows for scalability, making it easier to incorporate additional features or modules in the future.

4.2 FRONTEND IMPLEMENTATION
The front end of the system is implemented using React along with Vite and CSS Modules to create a responsive, dynamic, and user-friendly interface. The implementation focuses on delivering an interactive experience for users, enabling them to browse products, view details, submit orders or quotes, and track their interactions seamlessly. Components are modular and reusable, which allows for easier maintenance and future updates.

4.3 BACKEND IMPLEMENTATION
The back end of the system is implemented using Node.js and Express.js, providing a robust and secure environment for server-side operations. It handles all core business logic, including data processing, API endpoints, authentication, and integration with external services such as email notifications and PDF generation. The backend communicates with the database, built using MongoDB, to store and retrieve information efficiently, ensuring data consistency and reliability across the system.
RESTful APIs are developed to facilitate seamless communication between the frontend and backend modules. These APIs handle requests such as user management, product updates, order processing, and analytics tracking. The backend also includes middleware for authentication, error handling, and input validation, enhancing security and system stability. Services such as quote intelligence and analytics are implemented on the server side, allowing for real-time data processing and generation of insights that support informed decision-making.
Overall, the backend implementation ensures that the system operates efficiently, securely, and reliably, providing a foundation for scalable growth. It manages the flow of data between modules, supports automated business processes, and enables integration with additional services, ensuring the system meets the functional and performance requirements of modern business operations.
 
4.4 DATABASE IMPLEMENTATION
The database is implemented using MongoDB, a NoSQL document-oriented database, to provide flexible and efficient data storage for the system. Collections are created to manage different types of information, including users, products, orders, quotes, and analytics data. Each collection is structured with relevant fields to store necessary attributes, such as user credentials, product details, order status, quotation information, and interaction metrics. References between collections are established to maintain relationships and ensure data consistency, such as linking orders and quotes to specific users and products.
Indexes are applied to frequently accessed fields to improve query performance, enabling fast retrieval of data even as the database grows. CRUD (Create, Read, Update, Delete) operations are implemented through the backend APIs, allowing seamless interaction between the application and the database. The database design also supports scalability, making it possible to accommodate increasing data volumes as the system expands.
By implementing a well-structured and optimized database, the system ensures secure, reliable, and efficient storage and retrieval of data. This enables smooth integration with the backend and frontend modules, supports automated processes, and allows real-time analytics and reporting, enhancing overall system performance.

The AI analytics module is implemented to provide intelligent insights and data-driven decision-making capabilities within the system. This module leverages machine learning algorithms to analyse historical data, such as user interactions, product popularity, order patterns, and quotations. The module is integrated with the backend through API endpoints, allowing real-time access to updated data from the database. It processes incoming data continuously, updating dashboards and generating reports that provide actionable insights for administrators and decision-makers. Features such as predictive analytics for quote intelligence, trend analysis for products, and performance metrics for user engagement are all managed within this module.
By implementing the AI analytics module, the system not only automates the generation of insights but also supports proactive decision-making. This enhances operational efficiency, improves customer satisfaction, and provides a competitive advantage by enabling management to make informed, timely, and strategic business decisions based on accurate data analysis.

Server.js
require('dotenv').config();
const express = require('express');
const mongoose = require('mongoose');
const cors = require('cors');
const http = require('http');
const { Server } = require('socket.io');

const quotesRouter = require('./routes/quotes');

const app = express();
const server = http.createServer(app);

// Socket.IO setup for real-time updates
const io = new Server(server, {
  cors: {
    origin: [
      process.env.FRONTEND_URL || 'http://localhost:5173',
      process.env.ADMIN_URL || 'http://localhost:5174'
    ],
    methods: ['GET', 'POST']
  }
});

// Make io accessible to routes
app.set('io', io);

// Middleware
app.use(cors({
  origin: [
    process.env.FRONTEND_URL || 'http://localhost:5173',
    process.env.ADMIN_URL || 'http://localhost:5174'
  ],
  credentials: true
}));
app.use(express.json());

// MongoDB Connection
mongoose.connect(process.env.MONGODB_URI)
  .then(() => {
    console.log('✅ Connected to MongoDB');
  })
  .catch((err) => {
    console.error('❌ MongoDB connection error:', err);
    process.exit(1);
  });

// Socket.IO connection handling
io.on('connection', (socket) => {
  console.log('🔌 Client connected:', socket.id);
  
  socket.on('disconnect', () => {
    console.log('🔌 Client disconnected:', socket.id);
  });
});

// Routes
app.use('/api/quotes', quotesRouter);

// Health check
app.get('/api/health', (req, res) => {
  res.json({ 
    status: 'ok', 
    timestamp: new Date().toISOString(),
    mongodb: mongoose.connection.readyState === 1 ? 'connected' : 'disconnected'
  });
});

// Error handling middleware
app.use((err, req, res, next) => {
  console.error('Error:', err);
  res.status(500).json({
    success: false,
    message: 'Internal server error',
    error: process.env.NODE_ENV === 'development' ? err.message : undefined
  });
});

// Start server
const PORT = process.env.PORT || 5000;
server.listen(PORT, () => {
  console.log(`
🚀 Server running on port ${PORT}
📡 API: http://localhost:${PORT}/api
🔌 WebSocket: ws://localhost:${PORT}
  `);
});
